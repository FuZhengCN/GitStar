# GitStar 收藏列表页面设计

## 背景

插件已有收藏功能（`useFavorites` hook），用户可在 RepoCard 和 RepoHeader 中点击 ☆ 收藏项目，数据存于 `chrome.storage.local` 的 `gitstar-favorites` 键（`owner/repo` 字符串数组）。但缺少一个集中查看已收藏项目的页面，用户收藏后无处回顾。

## 目标

在 popup 中新增「我的收藏」页面，用户可查看、管理所有本地收藏的项目。

## 方案

新增独立路由 `#/favorites`，顶栏右侧加 ☆ 图标入口，收藏页复用现有 RepoCard 组件展示列表。

### 路由设计

| 路由 | 页面 | 说明 |
|------|------|------|
| `#/` | 首页 | 不变 |
| `#/project/:owner/:repo` | 详情页 | 不变 |
| `#/favorites` | 收藏列表（新增） | 展示所有本地收藏项目 |

### 导航流程

```
#/ (首页) → 点击顶栏 ☆ → #/favorites → 点击项目 → #/project/x/y → 浏览器后退 → #/favorites
```

wouter hash 路由天然保留浏览器历史，从详情页后退正确回到收藏页。

### 顶栏改动

标语「发现优质开源项目」完整保留。在标语右侧新增 ☆ 图标按钮：

- 首页/详情页：☆ 为白色（未激活态），点击跳转 `#/favorites`
- 收藏页：☆ 为金色 `#f59e0b`（激活态），表示当前在收藏页

### 收藏页布局

```
┌─ 蓝底顶栏（标语 + ★）─────────────┐
├─ 内容区 ──────────────────────────┤
│  ← 返回发现                       │
│  ★ 我的收藏 (N)                    │
│  ┌─ RepoCard ───────────────────┐ │
│  │ 头像  项目名          ★(已收藏)│ │
│  │       描述                    │ │
│  │       ★ stars  🔤 language   │ │
│  └──────────────────────────────┘ │
│  ┌─ RepoCard ───────────────────┐ │
│  │ ...                          │ │
│  └──────────────────────────────┘ │
│  (空收藏时显示空状态)              │
└───────────────────────────────────┘
```

- 面包屑：「← 返回发现」链接回 `#/`
- 标题：「★ 我的收藏 (N)」，N 为收藏数量
- 列表：复用 `RepoCard` 组件，点击 ☆ 取消收藏后从列表移除
- 空状态：无收藏时显示「暂无收藏项目」+ 引导文案

### 数据获取

收藏列表仅存储 `owner/repo` 字符串，展示需要的仓库元数据（名称、描述、Star 数等）需逐个调用 `getRepoInfo()`。

- 使用 `Promise.allSettled` 并发请求所有收藏项目的仓库信息
- 部分请求失败不影响其他项目的展示（失败项目跳过）
- 请求期间显示骨架屏
- 收藏数量为 0 时不发请求，直接展示空状态

### 修改文件

| 文件 | 改动 |
|------|------|
| `extension/popup.tsx` | 新增 `FavoritesPage` 组件 + `#/favorites` 路由；顶栏新增 ☆ 入口 |
| `extension/components/RepoCard.tsx` | 无改动，直接复用 |

### 不改的文件

- `extension/hooks/useFavorites.ts` — 功能不变
- `extension/lib/github.ts` — 纯 API 层，不变
- `extension/components/RepoList.tsx` — 收藏页直接使用 RepoCard，不复用 RepoList（需要处理空状态和 loading 不同）
- `extension/contents/github-sidebar.tsx` — 内容脚本不涉及

## 状态处理

| 状态 | 表现 |
|------|------|
| 加载中 | 骨架屏（5 个卡片占位） |
| 有收藏 | RepoCard 列表，☆ 金色实心 |
| 空收藏 | 空状态：「暂无收藏项目」+ 引导回首页探索 |
| 部分失败 | 忽略失败项目，展示成功获取的项目 |
| 全部失败 | ErrorState 组件，提供重试按钮 |

## 交互细节

- 在收藏页点击 ☆ 取消收藏，项目从列表立即移除（乐观更新）
- 在收藏页点击项目名，跳转 `#/project/:owner/:repo`
- 在详情页点击收藏/取消收藏，返回收藏页时列表反映最新状态（`useFavorites` 有 `chrome.storage.onChanged` 监听）
- 浏览器后退从详情页回到收藏页，不重新请求数据（`useStaleCache` 命中 `repo:*` 缓存）
