# GitStar 扩展详情页渐进式信息披露 — Design Spec

## Overview

将扩展详情页从当前平铺式布局（方案 C）升级为**渐进式信息披露**设计。默认展示紧凑的评估视图，用户可一键展开进入深度阅读模式。核心原则：同一页面两种密度，无需切换。

**约束：** Chrome popup 宽度 400px，内容视口高度上限 600px。

---

## 页面结构（5 层）

```
┌─ Layer 1: 身份卡片 ──────────────────────┐
│ ← 返回 / owner / repo                    │
│ [头像] repo/name                          │
│        描述（2行截断）                     │
└──────────────────────────────────────────┘
┌─ Layer 2: 核心操作区 ────────────────────┐
│ [★ Star 按钮]  230k stars                │
│                 🍴48k · 👀6.5k · JS · MIT │
└──────────────────────────────────────────┘
┌─ Layer 3: 辅助操作 ──────────────────────┐
│ [☆ 收藏]  [🔗 打开 GitHub]               │
└──────────────────────────────────────────┘
┌─ Layer 4: README 预览 / 全量 ────────────┐
│ 📖 README.md          12KB · ~5 min      │
│ ┌──────────────────────────────────────┐ │
│ │ (预览: 200px 渐变遮罩 / 全量: 展开)   │ │
│ └──────────────────────────────────────┘ │
│ [展开阅读全部 ↓]  /  [📋 目录] [收起 ▴] │
└──────────────────────────────────────────┘
┌─ Layer 5: 项目详情（可折叠） ────────────┐
│ [📋 项目详情 ▴]                          │
│ 创建时间 | 最新版本                       │
│ 更新时间 | 协议                           │
│ 🏷️ react · javascript · ui · frontend    │
└──────────────────────────────────────────┘
```

Layer 5 默认折叠，减少初始认知负荷。

---

## 双模式切换

### 预览模式（默认）

- 头部完整：面包屑 + 头像(32px) + 标题(13px bold) + 描述(11px, 2行截断)
- Star 操作区：蓝色实心 Star 按钮 + 统计指标
- 辅助操作：收藏 + 打开 GitHub（无复制链接、无 Token 标签）
- README 预览：前 60KB 渲染，max-height 200px，底部渐变遮罩
- 项目详情：折叠

### 阅读模式（展开后）

触发：点击 README 底部「展开阅读全部 ↓」

- **头部过渡**：完整头部 opacity 1→0，Mini-bar opacity 0→1（交叉淡入淡出 ~150ms）
- **README 展开**：max-height 移除，渐变遮罩消失（高度缓动 ~200ms）
- **Mini-bar**：position:sticky; top:60px（popup 顶栏下方 z-20），内容为 [小头像 24px] repo/name | [★ Star] [☆ 收藏]
- 目录按钮激活，点击弹出目录浮层（从 README h2/h3 自动生成）
- 底部「收起 README ▴」按钮 → scrollTo(0,0) + 回到预览模式

## Star 交互

### 三种状态

| 状态 | 按钮样式 | 触发条件 |
|------|---------|----------|
| 未 Star | `bg-[#3b82f6] text-white` 实心蓝底 | 默认（已检测未 Star） |
| 已 Star | `bg-[#f0fdf4] text-[#16a34a] border border-[#16a34a]` | 点击 Star 成功后 |
| 无 Token | `bg-[#f3f4f6] text-[#9ca3af] border-dashed` + 提示"配置 Token 后可 Star" | Token 未配置 |

### 过渡动画

1. Loading（~150ms）：按钮文字变 "..."，disabled
2. Success（~200ms）：背景色过渡到绿色，Star 数量 +1 并短暂高亮
3. Stable：绿色稳态，可再次点击 Unstar

### Unstar

- 已 Star 状态下点击直接取消，无确认弹框
- 数字 -1，回到蓝色未 Star 状态
- 与 GitHub 原生行为一致

---

## README 目录

- 从渲染后 DOM 的 h2/h3 元素自动提取
- 点击 README 标题栏「📋 目录」按钮弹出浮层
- 浮层 `shadow-[0_4px_12px_rgba(0,0,0,0.1)]`，点击锚点跳转后自动关闭
- 当前滚动位置对应的目录项高亮（`bg-[#eff6ff]`）

## 视觉参数

### 间距

| 位置 | 值 |
|------|-----|
| 内容区 → 顶栏 | `pt-[60px]` |
| 内容区 padding | `p-4` (16px) |
| 卡片之间 | `gap-2` (8px) |
| 卡片内部 | `p-3` (12px) |
| 卡片内元素间 | `gap-2.5` (10px) |
| 底部留白 | `pb-4`（无固定底栏） |

### 阴影层级

| Level | 值 | 用途 |
|-------|-----|------|
| 0 | `shadow-none` | 静态块内部分隔 |
| 1 | `shadow-[0_1px_4px_rgba(0,0,0,0.04)]` | 信息卡片 |
| 2 | `shadow-[0_1px_4px_rgba(0,0,0,0.06)]` | Mini-bar |
| 3 | `shadow-[0_4px_12px_rgba(0,0,0,0.1)]` | 浮层（目录） |

### 字体阶梯

| 用途 | 大小 | 字重 | 颜色 |
|------|------|------|------|
| 仓库名称 | 13px | bold | #1e1b4b |
| 描述 | 11px | normal | #6b7280 |
| Star 按钮 | 13px | semibold | white / #16a34a |
| 统计数字 | 12px | semibold | #1e1b4b / #f59e0b |
| 辅助按钮 | 11px | normal | #6b7280 |
| README 正文 | 12px | normal | #374151 |
| 面包屑 | 11px | normal | #3b82f6 |
| 辅助文本 | 10px | normal | #9ca3af |

### 配色（严格遵循项目配色表）

禁止使用 `#6366f1` / `#4f46e5`（indigo），统一用 `#3b82f6` / `#2563eb`。

## 组件变更

### 新增

- **MiniBar**：阅读模式 sticky 头部，含标题 + Star 按钮 + 收藏按钮
- **TocOverlay**：README 目录浮层，自动提取标题锚点

### 修改

- **DetailPage**：新增 `readmeExpanded` 状态控制双模式切换，头部/Mini-bar 交叉淡入淡出
- **RepoHeader**：移除 Token 状态标签、复制链接按钮。简化操作行（仅收藏 + 打开 GitHub）。头像从 40px 缩到 32px
- **ReadmeViewer**：标题栏新增「📋 目录」按钮和阅读时间估算。展开/收起动画

### 删除

- Token 状态标签（从面包屑行移除）
- 复制链接按钮（从操作行移除）

---

## Non-Goals

- 不改动 popup 顶栏（模式切换、收藏入口保持在顶栏）
- 不改动 Star API 调用逻辑（`starRepo`/`unstarRepo`/`checkStarred` 保持不变）
- 不新增依赖
- 不改动 README 解析/缓存逻辑（Worker 解析、60KB 预览限制保持不变）

---

## Open Risks

- **Mini-bar 与 popup 顶栏的 sticky 叠加**：需验证 `top:60px` 在不同 Chrome 版本中表现一致
- **交叉淡入淡出动画**：在 popup 有限视口内可能触发重排，需实测性能
- **目录提取**：依赖 README 渲染后的 DOM，需处理 Worker 异步解析的时序
